Загрузка процессов: От Тега к Каюте (Cabin)
В BoxOS мы полностью отказываемся от формата ELF и сложных заголовков. Процесс загрузки превращается из «анализа анкеты» в «предоставление пространства».

1. Концепция: Flat Image (Плоский образ)
   Вместо того чтобы хранить внутри файла инструкции по его сборке (секции, релокации, таблицы), исполняемый файл в BoxOS является прямым слепком памяти.

Формат: Чистый бинарный код (Flat Binary).

Точка входа: Фиксированная — 0x3000.

Метаданные: Отсутствуют внутри файла. Вся информация о правах и типе файла вынесена в Tagfs.

2. Системные Теги доступа
   Файл становится программой только тогда, когда система наделяет его соответствующим статусом через системные теги.

Тег Смысл Права
app Пользовательское приложение Доступ к стандартным Decks (Storage, Network, Ops).
utility Системная утилита Доступ к привилегированным опкодам System Deck (FF).
Важно: Если на файле нет одного из этих тегов, System Deck откажет в создании процесса. Файл останется просто набором данных.

3. Cabin (Каюта): Изоляция памяти
   Каждый процесс живет в своей Каюте — непрерывном и изолированном сегменте физической памяти.

Изоляция: Стены Каюты возводятся аппаратно (MMU). Процесс не может заглянуть в чужую Каюту или в коридоры ядра.

Фиксированный старт: Для кода внутри Каюты мир всегда начинается с одного и того же адреса.

Zero-Copy Loading: Ядро не парсит файл. Оно берет объект из Tagfs и одним DMA-трансфером копирует его в Каюту.

4. Алгоритм запуска (Spawn Flow)
   Когда поступает запрос notify() с префиксом FF:01 (Spawn Process):

Verification: System Deck проверяет ID объекта в базе Tagfs на наличие тега app или utility.

Allocation: Ядро выделяет пустую Каюту нужного размера.

Deployment: Тело объекта (бинарный код) копируется в Каюту, начиная с адреса исполнения.

IPC Setup: Настраиваются страницы 0x1000 (Notify) и 0x2000 (Result).

Ignition: Guide передает управление на 0x3000. Процесс «просыпается» уже внутри своей Каюты.

5. Почему это лучше ELF?
   Скорость: Время запуска программы = времени чтения данных с диска. Ноль секунд на парсинг заголовков.

Простота ядра: Лоадер в ядре занимает 15-20 строк кода вместо сотен.

Безопасность: Права на запуск отделены от содержимого файла. Вы не можете запустить вирус, просто скачав его; система должна явно «одобрить» его тегом.
