Оболочка BoxOS
Основная концепция

Система управления файлами основана на тегах (ключ:значение) и системных тегах (однословных). Каталоги отсутствуют.

Пример тегов:
project:boxos
status:urgent
type:document
author:john
trashed # системный тег
app # системный тег
utility #системный тег

Команды
Управление файлами
create <имя*файла> [теги] — Создать файл с тегами.
show <имя*файла> — Показать содержимое файла.
files [теги] — Вывести список файлов, соответствующих указанным тегам.
Формат вывода:

    Files
    <имя_файла>   [<тег1>, <тег2>, ...]
    tag <имя_файла> <ключ:значение> — Добавить тег к файлу.
    untag <имя_файла> <ключ:значение> — Удалить тег у файла.
    name <старое_имя> <новое_имя> — Переименовать файл.
    trash <имя_файла> — Переместить файл в корзину (добавляет системный тег trashed, теперь файл не видно в выборке).

    erase <имя_файла> — Полное удаление файла.
    erase trashed — Очистить корзину (полностью удалить все файлы с тегом trashed).

Рабочее пространство

    use [теги] — Установить контекст. Дальнейшие команды будут применяться только к файлам, содержащим все указанные теги. Контекст отображается в приглашении оболочки (например, [project:boxos] ~). Также этот контекст наследуют нужные процессы - ты помимо выборки файлов указываешь контест текущего процесса(набор тегов для него), что в дальнейшем повлияет на планировщик.

    use (без аргументов) — Сбросить контекст.

Системные утилиты

    me — Показать информацию о системе и пользователе.
    info <объект> — Вывести подробную информацию об объекте (например, о файле).
    edit <имя_файла> — Открыть файл в текстовом редакторе.
    say "<текст>" — Вывести текст.
    reboot — Перезагрузить систему.
    bye — Выключить систему.

Особенности работы с именами файлов

    Файлы с одинаковыми именами могут существовать, если у них разные теги.
    При создании файла с существующим именем система предложит его изменить. Отказ допустим.
    При обращении к файлу по неуникальному имени система запросит выбор конкретного файла из списка.

Пример сеанса работы:

~ create report.txt project:boxos status:active
~ tag report.txt author:alex
~ files project:boxos
Files
report.txt [project:boxos, status:active, author:alex]

~ use project:boxos
[project:boxos] ~ files
Files
report.txt [project:boxos, status:active, author:alex]

[project:boxos] ~ use
~
